# Revenue Forecast vs Actual UI Redesign Plan

## Current State Analysis

### Existing Pages (Fragmented)
1. **DashboardPage** - High-level metrics, monthly chart (actual vs expected)
2. **ExpectedRevenuePage** - Budget entry table
3. **ActualRevenuePage** - Actual tracking table
4. **ForecastPage** - Prediction algorithms with projections

### Problems with Current Design
- Data is scattered across 4 separate pages
- No unified view comparing expected vs actual at source level
- Variance analysis requires mental math
- No visual representation of performance by source
- Forecast page is disconnected from actual/expected tracking
- Monthly comparison requires switching between pages

## Implementation Plan

### Phase 1: Core Infrastructure
1. **Create new unified page component**: `RevenueIntelligencePage.tsx`
2. **Create tab navigation component**: Use shadcn Tabs
3. **Create shared data hook**: `useRevenueAnalytics` - consolidates all calculations
4. **Update types**: Add variance calculation types, forecast confidence types

### Phase 2: Overview Tab
1. **Stat cards row**: Reuse existing StatCard with enhanced calculations
2. **Combined trend chart**: Extend existing Recharts implementation
3. **Top/Bottom performers**: New component `PerformanceInsights.tsx`
4. **Quick insights**: New component `RevenueInsights.tsx`

### Phase 3: Monthly Analysis Tab
1. **Month selector**: New component with navigation
2. **Waterfall chart**: New Recharts implementation
3. **Source breakdown table**: Extend RevenueTable with variance columns
4. **Heatmap view**: New component `VarianceHeatmap.tsx`

### Phase 4: Source Performance Tab
1. **Source cards**: New component `SourceCard.tsx` with sparklines
2. **Filtering/search**: New filter bar component
3. **Expandable detail view**: Sheet or inline expansion
4. **Inline editing**: Integrate existing input patterns

### Phase 5: Forecast Tab
1. **Scenario controls**: New component `ScenarioControls.tsx`
2. **Area chart with bands**: Extend Recharts with confidence intervals
3. **Source-level forecasts**: Table with confidence scores
4. **Cash runway integration**: Pull from existing accounting context

### Phase 6: Polish & Integration
1. **Responsive design**: Ensure mobile-friendly layouts
2. **Animation & transitions**: Smooth tab switches, chart animations
3. **Keyboard navigation**: Accessibility
4. **Update navigation**: Replace old pages with new unified page
